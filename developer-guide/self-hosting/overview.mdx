# Self-Hosting Agentix

Plan and architect your Agentix deployment with confidence. This guide covers system requirements, architecture patterns, and essential considerations for enterprise-grade self-hosting.

<Warning>
Self-hosting requires careful planning of system requirements, network architecture, and security considerations. Review all sections before proceeding with deployment.
</Warning>

## Architecture Patterns

### Single-Zone Deployments

<CardGroup cols={2}>
<Card title="Standalone Server" icon="server">
  **Best for**: Development, testing, small teams (&lt;100 users)
  
  **Architecture**:
  - Single server with all components
  - Local database and cache
  - Basic load balancing
  - Standard firewall configuration
  
  **Limitations**:
  - Single point of failure
  - Limited scaling options
  - Regional performance constraints
</Card>

<Card title="Distributed Single-Zone" icon="network-wired">
  **Best for**: Production deployments (&lt;1,000 users)
  
  **Architecture**:
  - Separate application and database servers
  - Dedicated cache layer
  - Load balancer with SSL termination
  - Advanced firewall segmentation
  
  **Benefits**:
  - Independent scaling of components
  - Better fault isolation
  - Performance optimization per service
</Card>
</CardGroup>

### Enterprise Multi-Zone Architecture

For enterprise deployments requiring high availability, global distribution, and regulatory compliance:

```yaml
┌─────────────────────────────────────────────────────────┐
│                    Global Load Balancer                │
│                   (DNS-based routing)                  │
└─────────────────────┬───────────────────────────────────┘
                      │
        ┌─────────────┼─────────────┐
        │             │             │
   ┌────▼────┐   ┌────▼────┐   ┌────▼────┐
   │ Zone A  │   │ Zone B  │   │ Zone C  │
   │ Primary │   │ Active  │   │ DR/Cold │
   └─────────┘   └─────────┘   └─────────┘
```

<AccordionGroup>
<Accordion title="Multi-Zone Components">
**Zone A - Primary Region:**
- Primary database cluster (read/write)
- Full application stack
- Real-time data processing
- Primary user traffic

**Zone B - Active Secondary:**
- Read replica databases
- Full application stack
- Automated failover capability
- Regional user traffic

**Zone C - Disaster Recovery:**
- Cold standby infrastructure
- Backup storage systems
- Emergency activation procedures
- Data replication target
</Accordion>

<Accordion title="Inter-Zone Communication">
**Database Replication:**
- Streaming replication for real-time sync
- Point-in-time recovery capabilities
- Cross-zone backup validation

**Application Synchronization:**
- Session state replication
- Configuration synchronization
- Asset and media distribution

**Network Requirements:**
- Dedicated private connectivity (VPN/MPLS)
- Minimum 1Gbps inter-zone bandwidth
- &lt;50ms latency between zones
</Accordion>
</AccordionGroup>

## System Requirements

### Hardware Specifications

<Tabs>
<Tab title="Small Scale (Up to 500 users)">
**Single Server Requirements:**
- **CPU**: 8 cores (3.0GHz+ Intel/AMD)
- **RAM**: 16GB DDR4 ECC
- **Storage**: 500GB NVMe SSD (enterprise grade)
- **Network**: 5Gbps dedicated connection
- **Redundancy**: UPS, RAID 1 for OS

**Recommended Specs:**
- **CPU**: Intel Xeon or AMD EPYC
- **RAM**: 32GB for headroom
- **Storage**: 2x 1TB NVMe in RAID 1
- **Network**: Dual 10Gbps NICs

- **CPU**: 16 cores (3.5GHz+)
- **RAM**: 32GB DDR4 ECC

- **CPU**: 8 cores

- **Primary**: 32 cores, 128GB RAM, 4TB storage

**Infrastructure Services:**
- **Load Balancers**: 2+ redundant units

### Network Requirements

<AccordionGroup>
<Accordion title="Bandwidth Requirements">
**Per User Concurrent Load:**
- **API Traffic**: 50KB/s average, 200KB/s peak
- **WebSocket**: 10KB/s sustained
- **Voice Calls**: 64kbps (Opus codec)
- **File Transfers**: Variable, up to 10MB/file

**Scaling Calculations:**
- 100 users: ~10Mbps sustained, 40Mbps peak
- 500 users: ~50Mbps sustained, 200Mbps peak
- 1,000 users: ~100Mbps sustained, 400Mbps peak
- 5,000+ users: 500Mbps+ sustained, 2Gbps+ peak
</Accordion>

<Accordion title="Latency Requirements">
**Internal Communications:**
- **Database queries**: &lt;5ms local, &lt;20ms cross-zone
- **Cache operations**: &lt;1ms local, &lt;10ms cross-zone
- **Service mesh**: &lt;2ms between services
- **Load balancer**: &lt;5ms request routing

**External Communications:**
- **API responses**: &lt;100ms to end users
- **WebSocket messages**: &lt;50ms real-time
- **Voice calls**: &lt;150ms end-to-end
- **File downloads**: Based on bandwidth, not latency-critical
</Accordion>

<Accordion title="DNS and Service Discovery">
**DNS Requirements:**
- **Primary domain**: Agentix web interface
- **API subdomain**: RESTful API endpoints
- **WebSocket subdomain**: Real-time connections
- **CDN integration**: Static asset delivery

**Internal DNS:**
- **Service discovery**: For microservices architecture
- **Health checking**: Automated failover support
- **Geographic routing**: Multi-zone deployments
</Accordion>

## Local Configuration

### Basic Configuration File

Create a `config.yaml` file for local customization:

```yaml
# Basic local configuration
server:
  host: "0.0.0.0"  # Allow external access
  port: 7880
  
# Database (embedded SQLite for local use)
database:
  type: sqlite
  path: /var/lib/agentix/agentix.db
  
# Redis (embedded for local use)  
redis:
  embedded: true
  data_dir: /var/lib/agentix/redis

# RTC Configuration
rtc:
  enabled: true
  tcp_port: 7881
  port_range_start: 50000
  port_range_end: 60000
  # Set to your external IP if accessing from other machines
  external_ip: auto
  
# Logging
logging:
  level: info
  file: /var/log/agentix/server.log
```

### Environment Variables

Common environment variables for local setup:

```bash
# Core settings
AGENTIX_HOST=0.0.0.0
AGENTIX_PORT=7880
AGENTIX_LOG_LEVEL=info

# Database
AGENTIX_DB_TYPE=sqlite
AGENTIX_DB_PATH=/data/agentix.db

# Security (generate secure values for production)
AGENTIX_API_KEY=your_api_key_here
AGENTIX_JWT_SECRET=your_jwt_secret_here

# External access
AGENTIX_EXTERNAL_URL=http://your-ip-or-domain:7880
```

## Docker Compose Setup

Complete Docker Compose configuration for local development:

<CodeGroup>
```yaml docker-compose.yml
version: '3.8'

services:
  agentix:
    image: agentixhub/agentix:latest
    container_name: agentix-server
    restart: unless-stopped
    ports:
      - "7880:7880"      # API & Web UI
      - "7881:7881"      # RTC TCP
      - "50000-60000:50000-60000/udp"  # RTC UDP
    environment:
      - AGENTIX_CONFIG_PATH=/etc/agentix/config.yaml
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD:-agentix123}
      - REDIS_PASSWORD=${REDIS_PASSWORD:-redis123}
    volumes:
      - ./config:/etc/agentix:ro
      - agentix-data:/var/lib/agentix
      - agentix-logs:/var/log/agentix
    depends_on:
      postgres:
        condition: service_healthy
      redis:
        condition: service_started

  postgres:
    image: postgres:15-alpine
    container_name: agentix-postgres
    restart: unless-stopped
    environment:
      - POSTGRES_DB=agentix
      - POSTGRES_USER=agentix
      - POSTGRES_PASSWORD=${POSTGRES_PASSWORD:-agentix123}
    volumes:
      - postgres-data:/var/lib/postgresql/data
    ports:
      - "5432:5432"
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U agentix -d agentix"]
      interval: 10s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    container_name: agentix-redis
    restart: unless-stopped
    command: redis-server --requirepass ${REDIS_PASSWORD:-redis123}
    volumes:
      - redis-data:/data
    ports:
      - "6379:6379"

volumes:
  agentix-data:
  agentix-logs:
  postgres-data:
  redis-data:
```

```bash .env
# Environment variables for Docker Compose
POSTGRES_PASSWORD=secure_postgres_password
REDIS_PASSWORD=secure_redis_password
AGENTIX_EXTERNAL_URL=http://localhost:7880
```
</CodeGroup>

## Network Access

### Local Network Access

To access Agentix from other devices on your network:

<CodeGroup>
```bash Firewall (Ubuntu/Debian)
# Allow Agentix ports through firewall
sudo ufw allow 7880/tcp comment "Agentix API"
sudo ufw allow 7881/tcp comment "Agentix RTC TCP" 
sudo ufw allow 50000:60000/udp comment "Agentix RTC UDP"

# Check firewall status
sudo ufw status
```

```bash Firewall (CentOS/RHEL)
# Open ports in firewalld
sudo firewall-cmd --permanent --add-port=7880/tcp
sudo firewall-cmd --permanent --add-port=7881/tcp
sudo firewall-cmd --permanent --add-port=50000-60000/udp
sudo firewall-cmd --reload

# Check open ports
sudo firewall-cmd --list-ports
```

```bash Find Your IP Address
# Get your local IP address
ip addr show | grep "inet " | grep -v 127.0.0.1

# Or use hostname command
hostname -I
```
</CodeGroup>

Then access Agentix at: `http://YOUR_LOCAL_IP:7880`

### Port Forwarding (Router Setup)

For external internet access, configure your router:

| Service | Internal Port | External Port | Protocol | Description |
|---------|---------------|---------------|----------|-------------|
| Web/API | 7880 | 7880 | TCP | Main interface |
| RTC TCP | 7881 | 7881 | TCP | Fallback for RTC |
| RTC UDP | 50000-60000 | 50000-60000 | UDP | Media streams |

## Troubleshooting

### Common Issues

<AccordionGroup>
<Accordion title="Port Already in Use">
**Error**: `bind: address already in use`

**Solution**:
```bash
# Check what's using the port
sudo lsof -i :7880

# Kill the process if safe to do so
sudo kill -9 <PID>

# Or change the port in config
sed -i 's/7880/7881/g' config.yaml
```
</Accordion>

<Accordion title="Can't Access from Other Devices">
**Issue**: Agentix works on localhost but not from other devices

**Solutions**:
1. Check firewall settings (see Network Access section)
2. Ensure Agentix binds to `0.0.0.0` not `127.0.0.1`
3. Verify your local IP address
4. Test connectivity: `telnet YOUR_IP 7880`
</Accordion>

<Accordion title="RTC/Audio Issues">
**Issue**: Audio calls don't work or poor quality

**Solutions**:
1. Check UDP port range is open: `50000-60000`
2. Verify external IP detection:
   ```bash
   curl ifconfig.me  # Check your external IP
   ```
3. Update RTC configuration with correct external IP
4. Test with STUN server connectivity
</Accordion>
</AccordionGroup>
