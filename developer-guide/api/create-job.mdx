The Create Job API endpoint allows you to programmatically start new jobs (workflows) in Agentix. This is the primary integration point for external systems like CRMs, ticketing platforms and web applications.

<Info>
Jobs represent workflow instances that process customer interactions through your configured PowerLinks and agents.
</Info>

## Authentication

All API requests require authentication using your organization's API key.

<CodeGroup>
```bash Headers
x-api-key: YOUR_API_KEY
Content-Type: application/json
```

```bash Get API Key
# Contact your organization administrator to get your Integration API key.
# Only organization administrators can generate API keys.
```
</CodeGroup>

## Endpoint

<CodeGroup>
```http Request
POST /v1/jobs
```

```bash Base URL
https://launch.agentixhub.ai/api
```
</CodeGroup>

## Request Parameters

### Body Parameters

<ParamField body="data" type="object" required>
  The job creation data object containing all workflow and customer information.

  <Expandable title="data properties" defaultOpen>
    <ParamField body="workflow" type="integer" required>
      The unique identifier of the workflow to execute. Get this from your cPanel → Workflows.
    </ParamField>

    <ParamField body="validUntil" type="integer" required>
      Time in seconds until the job request expires. Recommended range: 60-3600 seconds.
    </ParamField>

    <ParamField body="language" type="string" default="en">
      Language code for the conversation interface (e.g., "en", "ar", "fr").
    </ParamField>

    <ParamField body="customer" type="object">
      Customer information for the job.
      
      <Expandable title="customer properties">
        <ParamField body="name" type="string">
          Full name of the customer
        </ParamField>
        
        <ParamField body="mobile" type="string">
          Customer's mobile phone number in international format
        </ParamField>
        
        <ParamField body="email" type="string">
          Customer's email address
        </ParamField>
        
        <ParamField body="customerID" type="string">
          Your internal customer identifier or reference
        </ParamField>
      </Expandable>
    </ParamField>

    <ParamField body="externalRefNumber" type="string">
      External system reference number (CRM ID, ticket number, etc.)
    </ParamField>

    <ParamField body="metadata" type="string">
      Additional context or notes about the job
    </ParamField>

    <ParamField body="input" type="array">
      Array of input parameters for the workflow
      
      <Expandable title="input item properties">
        <ParamField body="key" type="string" required>
          Parameter key name
        </ParamField>
        
        <ParamField body="value" type="string" required>
          Parameter value
        </ParamField>
        
        <ParamField body="type" type="string" required>
          Data type (TEXT, NUMBER, EMAIL, PHONE, etc.)
        </ParamField>
        
        <ParamField body="description" type="string">
          Human-readable description of the parameter
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

try {
  const response = await createJob(jobData);

  if (error.response?.status === 400) {
    console.error('Invalid request:', error.response.data.error.message);
  } else if (error.response?.status === 429) {
    console.log('Rate limited, retrying after delay...');

  } else {
    console.error('Unexpected error:', error.message);
  }
}

<Accordion title="Input Validation" icon="shield-check">
Validate input data before sending to the API:

```javascript
function validateJobData(data) {
  const required = ['workflow', 'validUntil'];
  const missing = required.filter(field => !data[field]);
  
  if (missing.length > 0) {
    throw new Error(`Missing required fields: ${missing.join(', ')}`);
  }
  
  if (data.validUntil < 60 || data.validUntil > 3600) {
    throw new Error('validUntil must be between 60 and 3600 seconds');
  }
  
  return true;
}
```

## Rate Limits

<Warning>
API requests are rate-limited to ensure platform stability. Current limits are:

- **100 requests per hour** per API key
- **10 requests per minute** for burst traffic
- Rate limit headers are included in all responses
</Warning>
