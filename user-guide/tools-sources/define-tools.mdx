## Creating Custom Tools

### Tool Definition Structure

Every tool requires a clear definition with inputs, outputs and processing logic:

```yaml
tool_definition:
  name: "email_validator"
  description: "Validates email addresses and checks domain reputation"
  category: "validation"
  inputs:
    - name: "email"
      type: "string"
      required: true
      description: "Email address to validate"
    - name: "check_domain"
      type: "boolean"
      default: true
      description: "Whether to check domain reputation"
  outputs:
    - name: "is_valid"
      type: "boolean"
      description: "Whether the email is valid"
    - name: "domain_score"
      type: "integer"
      description: "Domain reputation score (0-100)"
```

## Tool Configuration

### Parameters and Variables

Define flexible tool parameters:

<Note>
Use environment variables for sensitive configuration values like API keys.
</Note>

```yaml
tool_config:
  parameters:
    api_endpoint: "${EXTERNAL_API_URL}"
    timeout: 30
    retry_attempts: 3
    output_format: "json"
  variables:
    max_results: 100
    include_metadata: true
```

### Error Handling

Implement robust error handling:

1. **Input Validation**: Verify all required parameters
2. **Graceful Degradation**: Handle API failures gracefully
3. **Retry Logic**: Implement exponential backoff for transient failures
4. **Error Reporting**: Provide detailed error messages

### Performance Optimization

Optimize tool performance:

- **Caching**: Cache frequently accessed data
- **Batch Processing**: Process multiple items together
- **Async Operations**: Use asynchronous processing where possible
- **Resource Limits**: Set appropriate timeout and memory limits

## Tool Testing

### Unit Testing

Test individual tool functions:

```python
def test_email_validator():
    result = email_validator("test@example.com", check_domain=True)
    assert result["is_valid"] == True
    assert result["domain_score"] > 50
```

### Integration Testing

Test tools within workflow contexts:

```yaml
test_workflow:
  steps:
    - tool: "email_validator"
      inputs:
        email: "{{ test_email }}"
      expected_outputs:
        is_valid: true
```

## Best Practices

### Tool Design Principles

<Warning>
Design tools to be reusable, reliable and well-documented for maximum value.
</Warning>

1. **Single Responsibility**: Each tool should have one clear purpose
2. **Clear Interfaces**: Well-defined inputs and outputs
3. **Error Resilience**: Graceful handling of edge cases
4. **Performance**: Efficient resource usage
5. **Documentation**: Comprehensive usage examples
6. **Versioning**: Backward compatibility considerations

### Security Guidelines

- Validate all inputs to prevent injection attacks
- Use secure authentication for external services
- Implement proper access controls
- Log security-relevant events
- Regular security audits and updates
