### Connection Configuration

```json
{
  "source_name": "customer_db",
  "type": "postgresql",
  "connection": {
    "host": "localhost",
    "port": 5432,
    "database": "customers",
    "username": "${DB_USER}",
    "password": "${DB_PASSWORD}",
    "ssl": true
  },
  "query_timeout": 30
}
```

## API Sources

### REST API Configuration

Set up connections to external REST APIs:

<Note>
Use environment variables for sensitive information like API keys and passwords.
</Note>

```yaml
api_sources:
  crm_api:
    base_url: "https://api.crm.example.com/v1"
    authentication:
      type: "bearer_token"
      token: "${CRM_API_TOKEN}"
    rate_limiting:
      requests_per_minute: 100
    timeout: 30
```

## File System Sources

### Cloud Storage

Connect to various cloud storage providers:

- **AWS S3**: Scalable object storage
- **Google Cloud Storage**: Enterprise cloud storage
- **Azure Blob Storage**: Microsoft cloud storage
- **Dropbox**: Team collaboration storage
- **OneDrive**: Microsoft productivity storage

### Local File Systems

Access local and network file systems:

```json
{
  "source_name": "document_archive",
  "type": "filesystem",
  "path": "/data/documents",
  "permissions": {
    "read": true,
    "write": false,
    "execute": false
  },
  "file_patterns": ["*.pdf", "*.docx", "*.txt"]
}
```

## Source Security

### Access Controls

Implement proper security measures:

1. **Encryption**: Data encryption in transit and at rest
2. **Access Policies**: Role-based access control
3. **Audit Logging**: Track all data access and usage
4. **Data Masking**: Protect sensitive information

### Compliance Considerations

<Warning>
Ensure all data sources comply with relevant privacy and security regulations.
</Warning>

- GDPR compliance for EU data
- HIPAA for healthcare information
- PCI DSS for payment data
- SOX for financial data

## Testing and Validation

### Connection Testing

Validate source configurations before deployment:

```bash
# Test database connection
agentix source test --name customer_db

# Validate API endpoint
agentix source validate --name crm_api

# Check file system access
agentix source verify --name document_archive
```

### Data Quality Checks

Ensure data integrity and quality:

- Schema validation
- Data type verification
- Null value handling
- Duplicate detection
- Referential integrity
